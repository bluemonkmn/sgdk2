Tutorial 1 - Functional Game Without Importing

This page describes some steps to get a simple game running in SGDK2 without importing any external data. This is a very simple game, but steps through a few features of SGDK2 that are interesting and important to understand in the process of learning SGDK2. Keep in mind that creating a game from scratch (without importing anything) may look significantly more complicated than importing some key pieces (especially sprite definitions), but in the interest of completeness, this tutorial will expose all the details of re-creating this data rather than simply importing it.
  1. Start with the new/default project (select "<Blank>" from the File->New menu if necessary).
  2. Create a new graphic sheet.
    1. Right-click on "Graphic Sheets" and select "New", or select "Graphic Sheets" and select "New Object" from the File menu or toolbar.
    2. Enter "My Graphics" as the sheet name.
    3. Click the "Add" button or press Enter a second time.
    4. Double-Click on the new "Edit Images" node that appeared under "My Graphics" in the tree, or select it and press F4 (or select "Properties" from the View menu).
    5. Select "Flood Fill" from the tools menu, or the paint bucket from the toolbar.
    6. Select dark blue as the "Solids" color from the lower-left corner of the color palette window in the graphics editor by left-clicking on it.
    7. Click in the magnified edit area at the upper left of the graphics editor.
    8. Click the "Store to Selected Cell(s)" button in the Cell Manager window. This will be our background tile.
    9. Select the next cell in the Cell Manager (the one to the right of the first), and click "Load Selected Cell(s)".
    10. Repeat the previous 5 steps with Green as the "Solids" color. This will be our solid "wall" tile.
    11. Select the custom tool button (this looks like a star by default), or select "Star" from the "Custom Tool" menu in the "Tools" menu.
    12. Select the outlined shapes button from the toolbar, or select "Draw Outlined Shapes" from the tools menu. This will ensure that the star we're about to draw will have an outline.
    13. Select the gradient fill button from the toolbar, or select "Gradient Fills" from the tools menu. This will make a smooth gradient star from the fill in the center to the outline color around the edge.
    14. Click the yellow color in the recently used colors list to set the solid color to yellow.
    15. Move the mouse to a coordinate of approximately "15.5,17" in the magnified view (the coordinate is displayed in the magnified view's title bar) and drag a star upward to about 15.5, 1.5. Since the last step in the process above was to load an empty cell, you should be doing this on a transparent background represented ad grey and white diamonds.
    16. Store the tile in the third cell by clicking "Store to selected Cell(s)". This will be the image for an item the player will pick up.
    17. Select the fourth cell (the next empty cell) and load it with "Load Selected Cells(s)".
    18. Click the drop-down arrow next to the custom tool button and select "Triangle" from it, or select "Triangle" from the "Custom Tool" menu in the "Tools" menu.
    19. Left-click on Red in the recent colors list to select it as the "Solids" color.
    20. Move the mouse to a coordinate of approximately "15.5, 20" in the magnified view and drag a red triangle shape upward to about 15.5, 3.
    21. Store the triangle in the fourth cell (the next empty cell) by clicking "Store to Selected Cell(s)". This will be the image for our player sprite.
    22. Close the Graphics editor.
  3. Create a frameset.
    1. Right-click on the "Framesets" folder and select "New", or select the "Framesets" folder and then select "New Object" from the file menu or toolbar.
    2. Replace the frameset name with "My Frameset".
    3. Select "My Graphics" from the "Graphic Sheet" dropdown list.
    4. Select the first (blue) frame by left-clicking on it.
    5. Hold shift and select the star by left-clicking on it. This should also select the green frame.
    6. Add the three graphics to the frameset by pressing the toolbar button with the image of the right arrow, or by pressing Ctrl+A, or by selecting "Add Selected Cells to Frameset" from the "Frameset" menu.
    7. Close the Frameset Editor.
  4. Add a rotating sprite in the same frameset.
    1. Right-click on "Sprite Definitions" and select "New", or select "Sprite Definitions" and then select "New Object" from the File menu or toolbar.
    2. Enter "Player" for the sprite definitions name.
    3. Select "Rotating Sprite Wizard" from the "Sprite Definition" menu.
    4. Select "My Graphics" from the Graphic Sheet dropdown list.
    5. Select the Red Triangle and click Next.
    6. Leave the default value of 72 states, and click Next.
    7. Leave the default values of 1 repeat and 0 mask alpha level and click Next.
    8. Enter a prefix of "Norm" and click Next.
    9. Select "My Frameset" from the "Frameset" dropdown list and click Next.
    10. Select "Use a fixed size for all states" and click Next.
    11. Leave the default values of 32 and 32 for width and height, and click Next.
    12. Click Finish.
    13. Close the Sprite Definition editor.
  5. Create a Tileset. You can't create maps or define solidity without tiles.
    1. Create a new Tileset in the "Tilesets" folder; you should know how to create new objects by now.
    2. Enter "My Tileset" for the Tileset name.
    3. Leave the default width and height of 32 and 32; leave the default frameset of "My Frameset".
    4. Close the Tileset Editor.
  6. Create a solid tile category.
    1. Create a new tile category in the "Tile Categories" folder.
    2. Enter "Solid" for the new category name.
    3. Select "My Tileset" from the "Tileset" dropdown list.
    4. Click edit to define which tiles from "My Tileset" are in this category (tiles from multiple tilesets can be in the same category).
    5. Select the green wall tile.
    6. Drag it into the bottom pane or select "Add to Category" from the "Tiles" menu.
    7. Close the Tile Category window.
    8. Close the Tile Category Name window.
  7. Create a Map. The project can't compile without a default map, and if the project can't compile, you can't create a solidity definition.
    1. Create a new Map object in the "Maps" folder.
    2. Change the name to "Level 1"
    3. Click Add or press Enter again.
  8. Set the startup map.
    1. Right-click on the top-level "Project" node in the tree and select "Edit", or select the "Project" node and press the F4 key or select "Properties" from the View menu.
    2. Select "Level 1" from the "StartMap" dropdown list.
    3. Close the Project Properties window.
  9. Create a solidity definition.
    1. Create a new Solidity Definition in the "Solidity" folder.
    2. Enter "Standard" for the name.
    3. Select "Solid" from the dropdown list in the first row of the "Category" column.
    4. Select "SolidTileShape" from the dropdown list in the first row of the "Shape" column.
    5. Close the "Solidity" window.
  10. Define the rules for the player sprite.
    1. Display the sprite definition editor for the "Player" sprite definition, by double-clicking it, right-clicking it and selecting "Edit", or selecting it and pressing F4 or selecting "Properties" from the view menu.
    2. Select the "Rules" tab.
    3. Add a new rule by selecting "Add Rule" from the "Sprite Definition" menu, or by selecting the New button from the toolbar in the Rules tab.
      1. Enter "If player is pressing left" as the rule name.
      2. Select "If" from the first dropdown list to indicate that the type of this rule is a condition.
      3. Select "IsInputPressed" from the next dropdown list.
      4. Select "SpriteBase.InputBits.Left" from the "Input" dropdown list.
      5. Select "false" from the "InitialOnly" dropdown list. This allows the player to keep turning while holding the left arrow.
    4. Add another new rule named "Turn left"
      1. Select "Do" from the first dropdown and "CalculateRotatedState" from the second dropdown.
      2. Select "(int)Sprites.Player.State.Norm0" in the "FirstState" combo box.
      3. Type "72" in the "StateCount" combo box. You may recall this is the number of states we created for the player.
      4. Type "state+1" into the "NewState" combo box. Adding to the state rotates it to the left, while subtracting rotates it to the right. Keep in mind that you could normally import a rotating sprite instead of having to know these parameters, but this tutorial will show you all the details.
      5. Select "state" from the "Output To" combo box. This ensures that, after the new state is calculated, it is set as the player sprite's current state. Note that we use CalculateRotatedState instead of simply adding or subtracting 1 to ensure that the state never goes outside the range of 0 to 71.
      6. Check the "End If/End While" checkbox to end the first rule's condition.
    5. Add a new rule named "If player is pressing right"
      1. Select "If" from the first dropdown and "IsInputPressed" from the next dropdown.
      2. Select "SpriteBase.InputBits.Right" from the "Input" dropdown list.
      3. Select "false" from the "InitialOnly" dropdown list.
    6. Add a new rule named "Turn right"
      1. Select "Do" from the first dropdown and "CalculateRotatedState" from the second dropdown.
      2. Select "(int)Sprites.Player.State.Norm0" in the "FirstState" combo box.
      3. Type "72" in the "StateCount" combo box.
      4. Type "state-1" into the "NewState" combo box. This will rotate to the previous state, which is one click to the right.
      5. Select "state" from the "Output To" combo box.
      6. Check the "End If/End While" checkbox to end the "If player is pressing right" condition.
    7. Add a new rule named "If the player is pressing up"
      1. Select "If" from the first dropdown and "IsInputPressed" from the next dropdown.
      2. Select "SpriteBase.InputBits.Up" from the "Input" dropdown list/
      3. Select "false" from the "InitialOnly" dropdown list.
    8. Add a new rule named "Accelerate"
      1. Select "Do" from the first dropdown and "PolarAccelerate" from the second dropdown.
      2. Enter "5" in the "Acceleration" combo box.
      3. Select "(int)Sprites.Player.State.Norm0" from the "FirstState" dropdown.
      4. Enter "72" in the "StateCount" combo box.
      5. Check the "End If/End While" checkbox.
    9. Add a new rule named "React to solidity"
      1. Select "Do" from the first dropdown and "ReactToSolid" from the second dropdown.
    10. Add a new rule named "Move"
      1. Select "Do" from the first dropdown and "MoveByVelocity" from the second dropdown.
    11. Add a new rule named "Inertia"
      1. Select "Do" from the first dropdown and "ReactToInertia" from the second dropdown.
      2. Select the "Parameters" tab.
      3. In the first row of the grid, type "My inertia". This adds a parameter to this sprite type that can be specified on each sprite instance in the map editor.
      4. Switch back to the "Rules" tab
      5. Select "My_inertia" from the "RetainPercentVertical" and "RetainPercentHorizontal" dropdown lists.
    12. Close the sprite definition editor.
  11. Create a new layer.
    1. Locate the "Layers" folder under "Level 1" in the project tree and create a new layer in it.
    2. Enter "My Layer" for the layer name.
    3. Note the deault selection of "My Tileset" for the Tileset, and the default values of 100 and 15 for the layer width and height.
    4. Click Add.
  12. Edit the layer
    1. The layer editor should appear automatically when the layer is added, but if you need to re-display it, you can locate the "Editor" node inside the new "My Layer" node in the project tree, and double-click it.
    2. Scroll to the bottom of the layer
    3. Select the green tile from the "Tiles" pane
    4. Draw a floor accross the bottom of this layer by moving the mouse to tile coordinates "0,14" and dragging a straight row of green tiles to the right edge of the window. The tile coordinate is displayed in the map editor's status bar.
    5. Scroll to the right and repeat the previous step until there is a green floor across the entire bottom of the map.
    6. Select the star tile from the Tiles pane.
    7. Place the star on the blue background somewhere. Notice that it has a black (actually transparent) background. If we want to create tiles like this on the same layer as the background, they need to have their own background. We'll do this in the next step, "Define a tile mapping."
    8. Select the "Sprites" tab.
    9. Select "Add Sprites".
    10. Select "<All>" from the dropdown list of sprite categories.
    11. Select the red triangle sprite.
    12. In the sprite properties pane at the bottom of the sprites pane, enter "1" into the "Priority" field, and press Enter. This ensures that the sprite will draw after the layer it's in instead of before or instead of interleaving with the tiles. You probably won't have to change the default value because it is already 1, but be aware of this value for controlling how sprites draw.
    13. Scroll to the left end of the map and click the sprite onto the layer's blue background somewhere. Notice that the sprite doesn't need to worry about the background because it is drawn independently of the tiles in the layer, rather than replacing tiles in the layer.
    14. Select "Select Sprites" from the sprites pane.
    15. Click on the sprite that was just added to select it.
    16. In the sprite properties pane, select "Standard" in the "Solidity" field. Defining the solidity on the individual sprite instance allows different instances of the same sprite definition to react differently to map solidity or allows the same instance to act differently at different times.
    17. In the sprite properties pane, enter "98" for "My inertia". Notice that this is the parameter added to the sprite definition for this sprite type.
    18. Close the map editor.
  13. Define a tile mapping.
    1. Double-click "My Tileset" in the Tilesets folder (or select it and open it with F4 or the Properties command).
    2. Select the star tile in the bottom pane and notice that its frameset index is #2 in the status bar.
    3. In order to replace this tile with one that has a background, we'll override tile number 2: Select "New Tile" from the Tileset menu or the toolbar next to the "Mapped Tiles" dropdown box.
    4. Select "Custom", enter "2" into the Tile Value field, and click OK.
    5. Select and drag the blue frame from the bottom pane to the top pane and drop it on the left side of the star. Alternatively, you can select the blue tile in the bottom frame and press Ctrl-A to add it to the tile. If it gets added after the star, then select the star frame in the bottom pane and add that (with Ctrl-A) too. Then select the first star in the top pane and press the delete key to delete that frame from the tile.
    6. With only the blue frame in the top pane selected, replace the default value of "1" in the "Repeat Count" field with a "0". This causes the blue background frame to be merged with the star frame so they display at the same time.
    7. Feel free to go back to the map editor (double-click on "Editor" under the "My Layer" node in the project tree) and verify that the star has a blue background now.
    8. Close the Tileset Editor.
  14. Add a new plan to select a player sprite (map keyboard input to the sprite and scroll it into view).
    1. Add a new plan to the "Plans" folder under Maps - Level 1 - Layers - My Layer.
    2. Enter the name "Manage player sprite" in the Name field of the plan.
    3. Select "Add New Rule" from the "Plan" menu or the toolbar above the rule list.
      1. Enter the rule name "Map input to player sprite"
      2. Select "Do" from the first dropdown and "MapPlayerToInputs" from the second dropdown.
      3. Enter 1 in the PlayerNumber field.
      4. Select m_ParentLayer.m_Player_1 from the Target dropdown list.
    4. Add another new rule named "Scroll player sprite into view". Note: this could be done on the sprite definition instead (so you don't have to repeat this on every map) but if you want multiple players, it's better to see that it can be done here too.
      1. Select "Do" from the first dropdown and "ScrollSpriteIntoView" from the second.
      2. Select "m_ParentLayer.m_Player_1" from the "Sprite" dropdown list.
      3. Select "true" from the "UseScrollMargins" dropdown list.
    5. Close the plan editing window.
  15. Set the appropriate scrolling size for the map.
    1. Open the properties window for Level 1 (double-click on "Level 1" in the project tree)
    2. Click the "Wizard..." button.
    3. Click Next.
    4. Check the box for "My Layer" and click Next.
    5. Click Finish.
    6. Click Update.
  16. Press F5 or select "Run Project in Debug Mode" from the File menu

At this point you should be able to move the player around using the arrow keys and see the map scroll (although you might not notice it scrolling until you get to the star because the graphics are so uniform). If you'd like to further enhance the game to pick up the star, add some walls and introduce gravity, you can proceed with the steps below. After each step you can run the game again to see your progress (if any is visible) by pressing F5. Note that you will be un-able to re-compile (re-run) your game if it's still running, so it's a good idea to quit before proceeding to the next step.

  1. Close the running game window.
  2. Add a rule for gravity.
    1. Open the "Player" sprite definition for editing.
    2. Go to the Rules tab.
    3. Expand "If the player is pressing up" and select "Accelerate". This is to ensure that the new rule gets added in the right position (right before "React to solidity"). New rules are always added below the selected rule. Alternatively you could simply create the rule and then use the up and down arrow buttons on the toolbar to position it.
    4. Add a new rule before "React to solidity" named "Gravity"
      1. Select "Do" from the first dropdown and "AlterYVelocity" from the second dropdown.
      2. Enter 0.1 in the "delta" field.
    5. Close the sprite definition editor
  3. Create a counter to represent collected stars.
    1. Create a new counter in the "Counters" folder.
    2. Enter the name "Stars".
    3. Enter a maximum of 18.
    4. Leave the default Value of 1. This will allow you to see/test the inventory display as soon as it's implemented instead of waiting until the player has the ability to pick up stars.
    5. Close the Edit Counter window.
  4. Create an overlay map to display inventory.
    1. Create a new map in the "Maps" folder and name it "My Overlay".
    2. Remember to click the "Add" button after entering the name to add the map.
    3. Create a new layer in the new map's "Layers" folder named "Inventory".
    4. Before adding the layer, click the "Wizard..." button.
    5. Accept the default of "My Tileset" and click Next.
    6. Accept the default of 1 byte per tile and click Next.
    7. Check the box to delete all tiles and use a new background (this is required when adding a new layer). Select the first red triangle and click Next. We don't have a transparent background tile but we need one. We'll go back and replace the first red triangle tile with an empty tile in a moment.
    8. Accept the default scroll rate and click Next.
    9. At this point in the wizard there is sometimes a step that would allow you to position the layer within the map with a helpful preview window. However, since this layer is new and has no tiles yet, no preview is possible, and this step is skipped.
    10. Accept the default priority and click Next.
    11. Check the box to "Size the layer to fill the map's scrollable area." and click Next.
    12. Accept the default virtual size and click next.
    13. Click Finish.
    14. Click "Add" to add the layer.
    15. The layer full of red triangles will be displayed. We'll replace these with an empty tile in the next step. Close the layer editor.
  5. Define an empty tile
    1. Open "My Tileset" for editing.
    2. Add a new tile mapping (select "New Tile" from the Tileset menu).
    3. Select "Custom", enter "3" as the value and click OK.
    4. Select the triangle frame in the top "Frame Properties" pane.
    5. Press the delete key (or select "Delete Selected Frames" from the Tileset menu) and confirm the delete operation. Note that a tile without any frames is somewhat more efficient than a tile with a graphic cell that is all transparent.
    6. Close the Tileset Editor.
  6. Activate the overlay map as an overlay.
    1. Double-click on the top level "Project" node to open the project properties window.
    2. Select "My Overlay" from the "OverlayMap" dropdown list.
    3. Close the Project Properties window.
  7. Draw the Stars counter on the overlay map.
    1. Open the map editor for the "Inventory" layer of "My Overlay".
    2. Select the "Plans" tab.
    3. Add a new plan by clicking the "Add a New Plan" toolbar button inside the "Plans" frame.
    4. Select the plan that was added.
    5. Select "Snap to Tiles" from the Edit menu.
    6. Move the mouse to tile coordinate 1,1 (see status bar).
    7. Left click to add a point.
    8. Move the mouse to tile coordinate 19,2. Note that the size of the rectangle combined with the maximum value of the counter has an effect on how the inventory will be drawn. Notice that the maximum for stars was set to 18, and 18 is also the width (in tiles) of the rectangle.
    9. Left click to define a rectangle where stars will be drawn. Note that after you click the rectangle will go away and propose a third coordinate as you move the mouse around, but if you do not add another point and do move the mouse out of the map editor the rectangle will re-appear demonstrating that the plan in its saved form has two points defining a rectangle.
    10. Enter the name "Draw Stars" for the plan in the plan properties pane at the bottom left, and press Enter.
    11. Click the "Edit Selected Plan" button in toolbar in the "Plans" frame, or select "Selected Plan Details" from the Edit menu.
    12. Add a rule named "Draw stars at top of screen". (Click toolbar button to add a new rule or select "Add New Rule" from the "Plan" menu.)
    13. Select "Do" from the first dropdown and "DrawCounterAsTile" from the second dropdown.
    14. Enter 2 for the TileIndex parameter.
    15. Select "Counter.Stars" from the "Counter" dropdown list.
    16. Select "PlanBase.DrawStyle.RepeatRightToCounter" from the "style" dropdown list. This style will draw a bar filled with un-stretched (normal-size) stars and cut it off based not on the number of stars displayed, but on the ratios of owned stars to maximum stars and drawn bar size to total bar size. For example, if your bar were 3 tiles wide, and your max quantity were 2, but you only owned 1, then the bar would draw one and a half tiles because 1 is half of 2 and half of 3 is one and a half.
    17. Close the Plan Editor.
    18. Close the Map Editor.
  8. Define a category of tiles that should react to the player.
    1. Create a new category in the "Tile Categories" folder in the project tree.
    2. Enter the category name "Touchable".
    3. Select "My Tileset" and click "Edit...".
    4. Add the star tile to the category (drag it down or select it and select "Add to category" from the Tiles menu.
    5. Close the Tile Category window.
    6. Close the Tile Category Name window.
    7. Press the F7 key or select "Generate Project" from the file menu. If you don't do this, the new category might not show up as an available option when defining the sprite rule. If that happens, remember that F7 will re-compile the project, making all the recent changes available to the environment for "reflection" (reflection is a term used often in .NET programming that refers to the kind of process that SGDK2 uses to examine the generated code to see what objects are in a compiled project).
  9. Define a rule to allow player sprites to acquire stars.
    1. Open the "Player" Sprite Definition for editing.
    2. Go to the Rules tab.
    3. Select "Move".
    4. Add a new rule called "Touch tiles". Touch tiles is required once if the sprite interacts with tiles based on their category or tile numbers rather than simply their solidity. It creates a list of tiles that the sprite is touching that can be processed by other rules.
      1. Select "Do" in the first dropdown and "TouchTiles" in the second dropdown.
      2. Select "TileCategoryName.Touchable" in the "Category" dropdown list.
    5. Add a new rule called "Get stars".
      1. Select "Do" from the first dropdown and "TileTake" from the second dropdown.
      2. Enter the value 2 into the TileValue combo box. Remember that 2 was the number of the star tile we defined (with the blue background) in the TileSet editor.
      3. Select "Counter.Stars" from the "Counter" dropdown list.
      4. Enter "0" (zero) in the "NewValue" combo box. This indicates that when a star tile is removed, it should be replaced with tile number 0 (the empty background tile).
      5. Leave "Output to" empty.
    6. Close the sprite definition editor.
  10. Add walls to the map.
    1. Open "My Layer" from map "Level 1" in the editor.
    2. Select the green tile.
    3. Draw some walls between the player and the star. (Don't block the player completely of course.)
    4. Add some more stars if you like.
    5. Close the map editor.
  11. Run the project (press F5).
